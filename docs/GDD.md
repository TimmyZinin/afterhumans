## 1. Жанр и форма

**Жанр:** *narrative walker* (also known as walking simulator, first-person narrative experience).
**Аналоги:** Dear Esther, Firewatch, Gone Home, Tacoma, What Remains of Edith Finch, Everybody's Gone to the Rapture, Fugue in Void.
**НЕ:** action, RPG, puzzle, horror, survival, crafting, roguelike.

**Форма:** 3D от первого лица, линейная прогрессия через три сцены, ~10-15 минут прохождения.

---

## 2. Core gameplay loop

```
    [идти] → [встретить NPC] → [нажать E] → [прочитать диалог]
        ↓                          ↓
    [осмотреться]             [выбрать реплику]
        ↓                          ↓
    [продвинуться]           [получить знание]
        ↓                          ↓
    [сменить локацию] ← ← ← ← ← ← ← ←
        ↓
    [финальный выбор у Курсора]
```

Игрок в каждый момент времени делает одно из четырёх:
1. **Идёт** (WASD + мышь)
2. **Смотрит** (мышь)
3. **Взаимодействует** (E на NPC, предмете, Кафке)
4. **Выбирает реплику** (в открытом диалоге — цифры 1-5 или мышь)

Никаких боёв, инвентаря, прыжков, бега за врагами, сбора предметов. Один instrument — **ходить и говорить**.

## 3. Управление (PC, клавиатура+мышь)

| Действие | Клавиша | Описание |
|---|---|---|
| Движение вперёд | W | Ходьба вперёд |
| Движение назад | S | Ходьба назад |
| Движение влево | A | Шаг влево |
| Движение вправо | D | Шаг вправо |
| Смотреть | Мышь | Поворот камеры, FOV 65° |
| Взаимодействие | E | Говорить с NPC, подобрать записку, нажать Enter в Курсоре |
| Пропустить реплику | Space / E | Пропустить typewriter-анимацию текста, показать всю реплику сразу |
| Выбрать реплику | 1-5 / ЛКМ | Выбор диалоговой опции |
| Идти быстрее | Shift | Ускоренная ходьба (не бег — просто быстрее) |
| Пауза | Esc | Меню паузы |
| Полноэкранный / оконный | Alt+Enter | Стандартный |

**Чего нет:**
- Нет прыжка (Space пропускает текст, а не прыгает)
- Нет приседания
- Нет инвентаря
- Нет quest journal
- Нет мини-карты
- Нет HUD в играемом пространстве (кроме interaction prompt)

## 4. Камера

*(Детали в `ART_BIBLE.md` раздел 6.)*

**Ключевые параметры:**
- First-person, FOV 65°
- Высота 1.65m
- Head bob: middle в Ботанике, minimal в Городе, сильный в Пустыне
- Mouse smoothing: lag 0.1 (мягкость)
- Sensitivity: настраивается в меню (default 1.0)

**Cinematic moments:**
- Пробуждение в кресле: fade from black + slight drift
- Первый выход из Ботаники: 0.5 сек фиксированный widescreen шот коридора
- Первый взгляд на Город: замедление и пан
- Первый взгляд на Пустыню: slow pan справа налево по горизонту
- Приближение к Курсору: FOV сужается с 65° до 50° плавно за 30м

## 5. Система взаимодействия (Interaction System)

### Механика

1. Игрок подходит к интерактивному объекту (NPC, Кафка, записка, курсор)
2. На расстоянии ≤2m показывается **prompt** в worldspace: `[E] говорить` / `[E] взять` / `[E] взаимодействовать`
3. Игрок нажимает E → запускается соответствующий knot в Ink-скрипте или событие
4. После завершения взаимодействия — игрок возвращается к свободному движению

### Компонент `Interactable.cs`

Каждый interactive объект в сцене имеет компонент `Interactable` с полями:
- `string knotName` — имя knot'а в `dataland.ink` для запуска
- `string promptText` — текст над объектом (например, «говорить»)
- `float interactRadius` — дистанция активации (default 2.0m)
- `bool oneTime` — false для NPC (можно вернуться), true для записки (один раз)
- `UnityEvent onInteracted` — опциональные события при нажатии E

### Приоритет взаимодействия
Если в радиусе 2m несколько интерактивных объектов — выбирается **ближайший** к centre камеры (raycast по forward vector). Prompt показывается только для выбранного.

## 6. Система диалогов

### Движок
**Ink (inkle)** + `inkle/ink-unity-integration` (бесплатно, MIT).

### Поток
1. E на NPC → `story.ChoosePathString(knotName)` → запускается knot
2. Ink выводит текст → Unity рендерит субтитры через TextMeshPro с typewriter effect
3. Когда текст закончился — если есть `+ [choice]` → показываются варианты
4. Игрок выбирает → `story.ChooseChoiceIndex(i)` → Ink продолжает
5. Когда встречается `-> END` — диалог завершён, игрок возвращается к движению

### UI диалога
- Нижняя треть экрана
- Имя NPC (опционально — в нашем сценарии имена в тексте реплик)
- Текст с typewriter effect (30 chars/sec, можно пропустить)
- До 5 вариантов ответа снизу (префикс `> `)
- Subtle background (тёмная плашка с 60% opacity)

### Субтитры описаний
Ремарки в скобках (*Кафка виляет хвостом*) рендерятся курсивом и серым цветом, чтобы отличаться от прямой речи.

### State tracking
Все variables из `dataland.ink` автоматически сохраняются в памяти story object. На случай повторного прохождения — добавим save/load в Блоке 3.

## 7. Walking mechanics

### Базовый FirstPerson controller
**Unity Starter Assets FirstPerson** (бесплатный пакет от Unity).

### Параметры движения
| Параметр | Значение |
|---|---|
| Walk speed | 2.0 m/s |
| Run speed (Shift) | 3.5 m/s |
| Acceleration | 10 m/s² |
| Deceleration | 15 m/s² (быстрее стоп) |
| Turn speed | instant (мышью) |
| Gravity | 9.8 m/s² (на случай уклонов) |

### Коллизия
- CharacterController компонент
- Радиус 0.4m, высота 1.8m
- Slope limit 45°
- Не может проваливаться сквозь объекты

### Зоны без Shift (cinematic)
В определённых местах Shift временно отключается для поддержания темпа:
- При первом выходе в каждую локацию (~3 секунды)
- При приближении к Курсору (~20м до цели)
- В момент диалога с Николаем — Shift off на всё время разговора

## 8. Scene transitions

### Механика
Три отдельных Unity-сцены:
- `Scene_Botanika.unity`
- `Scene_City.unity`
- `Scene_Desert.unity`

### Переход между сценами
Trigger-коллайдер у двери/ворот → при входе игрока → fade to black (0.8s) → `SceneManager.LoadScene(...)` → fade from black (0.8s) → игрок спавнится в стартовой позиции новой сцены.

### State persistence
- Variables из Ink → сохраняются через singleton `DialogueManager.Instance.story` (не уничтожается между сценами)
- Кафка → persistent объект с `DontDestroyOnLoad`, перемещается в новую сцену автоматически
- Прогресс игрока (какая сцена текущая) → PlayerPrefs для возможного save/load

### Правило «нет возврата»
- Из Ботаники в Город — можно вернуться, пока не вошёл в Пустыню
- Из Пустыни обратно — **невозможно** (сюжетный принцип, технически — через deletion trigger у ворот в пустыню)

## 9. Fail states и sanity checks

### Fail states
**Нет.**
- Нельзя умереть
- Нельзя застрять
- Нельзя «сделать неправильно» — каждый выбор ведёт дальше
- Нельзя завалить диалог

### Sanity checks (тех-гарантии)
- Если игрок идёт в дверь города до разговора с Николаем — дверь выдаёт голос *«Не сейчас»* и не пускает
- Если игрок обходит NPC без разговора — они всё ещё доступны для возврата
- Если игрок случайно «сломал» Ink-стейт (редчайший случай) — fallback на последнюю сохранённую точку
- Если игрок идёт в пустыню без Кафки рядом (не должно случаться, но на всякий случай) — её позиция форсировано обновляется при переходе через ворота

## 10. Прогрессия

### Единственная метрика прогресса — **пройденные локации**
- **Stage 1:** Ботаника (0-6 мин)

## 11. Системы сохранения

### Episode 0 — минимальное
- Автосохранение при каждом переходе между сценами
- Автосохранение при каждом изменении Ink-variable
- Сохранение в PlayerPrefs + отдельный JSON в `Application.persistentDataPath/afterhumans_save.json`
- Одна ячейка сохранения (не нужно больше для 15-минутного эпизода)

### Continue / New Game
- При запуске игры в меню: если есть сохранение → *«Продолжить»*, иначе только *«Начать»*
- *«Начать заново»* перезаписывает сохранение без подтверждения

## 12. Accessibility

### Базовый минимум Episode 0
- Настройка яркости экрана в меню
- Настройка громкости музыки / SFX / диалогов (отдельно)
- Настройка mouse sensitivity
- Возможность пропустить typewriter effect (Space или E)
- Возможность отключить head bob (для motion sickness)
- Возможность отключить film grain (для чувствительных к шуму)

### Stretch (если время позволит)
- Настраиваемый размер шрифта субтитров
- Dyslexia-friendly font (OpenDyslexic — альтернатива IBM Plex Sans)
- Colorblind mode (deuteranopia, protanopia) через URP tint

## 13. Производительность (performance targets)

### Target hardware
- **MacBook Pro 13" M1 2020, 8 GB RAM** (Тим, primary)
- **Любой Apple Silicon Mac с 8+ GB** (друзья Тима)
- **Intel Mac с dedicated GPU + 16 GB** (потенциальные внешние игроки)

### Frame rate target
- **60 FPS** в Ботанике и Городе (небольшие сцены, оптимизация реальна)
- **30-45 FPS** в Пустыне (большая terrain scene, Megascans-ассеты)
- Минимум **24 FPS** в любой сцене, иначе оптимизация обязательна

### Memory budget
- RAM: **< 4 GB** unity process (чтобы не задавить Mac 8GB)
- VRAM: **< 2 GB** (M1 shared memory)
- Disk: **< 2 GB** итоговый билд

### Оптимизации при необходимости
- Occlusion culling для Ботаники и Города
- LOD groups для Megascans-ассетов в пустыне
- Texture compression (ASTC для Apple Silicon)
- Lightmap baking (statics)
- Disable real-time shadows для distant lights
- Reduce volumetric fog resolution

## 14. Sound design principles

*(Полные детали в `ART_BIBLE.md` раздел 8.)*

**Правило №1:** тишина — тоже звук. Не заполняй моменты пустоты фоновым ambientom ради «чтобы не было тихо».
**Правило №2:** каждое действие игрока имеет звук (шаги, открытие двери, E на NPC, выбор реплики).
**Правило №3:** музыка вступает в ключевые моменты, а не играет постоянно.

## 15. Playtest checklist

*Для использования после Блока 2 Production, перед релизом.*

### Core flow
- [ ] Игрок просыпается в Ботанике, видит Кафку, встаёт, ходит
- [ ] Игрок может подойти ко всем 5 NPC в Ботанике и поговорить
- [ ] Диалог с Николаем разблокирует дверь в город
- [ ] Переход в Город работает без багов
- [ ] Анна реагирует на подход Кафки (trigger работает)
- [ ] Смотрительница выпускает в пустыню
- [ ] Пустыня: ходьба длится ~2-3 минуты, Кафка следует
- [ ] Сервер: Кафка рычит, E на сервере издаёт глитч-звук
- [ ] Курсор: 5 вариантов ввода, каждый даёт свой финальный текст
- [ ] Титры + sting отображаются корректно
- [ ] Возврат в меню после финала работает

### Полировка
- [ ] Ни один NPC не заблокирован за стеной / в воздухе
- [ ] Ни один текст не обрезан / не переполняет экран
- [ ] Ни один звук не «ломается» (распадается, скрипит)
- [ ] Кафка не проходит сквозь объекты / не зависает
- [ ] FPS ≥ 30 везде
- [ ] Диалоги выглядят читаемо на 13" экране
- [ ] Save/load работает корректно (continue после выхода)

### Эмоциональные биты
- [ ] Момент Анны с Кафкой: триггер срабатывает, эмоционально попадает
- [ ] Николай: игрок понимает устройство Прогноза
- [ ] Финальный момент у Курсора: игрок чувствует важность выбора
- [ ] Credits: игрок досматривает до конца
